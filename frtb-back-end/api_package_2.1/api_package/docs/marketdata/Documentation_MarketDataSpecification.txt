OIS Discounting Curve Market Data Specification

OIS Discounting Curve supported currencies and curve names:
(1) USD - Curve_USD_OIS
(2) EUR - Curve_EUR_OIS
(3) GBP - Curve_GBP_OIS
(4) JPY - Curve_JPY_OIS
(5) CNY - Curve_CNY_Repo7D

All OIS Discounting Curves are following the same market data specification. We take 'Curve_USD_OIS' as an example for details.

---------------------------------

Curve_USD_OIS Specification

Market Data Type: 
Curve_USD_OIS

Description: 
The OIS Discounting Curve of USD currency

Market Data required headers:
MarketDataType - The market data object to construct with the market data.
Source - The field to store the name of market data source.
Revision - The field to store which price type of "Bid", "Ask" or "Mid" the market data is.
Currency - The currency name.
InstrumentType - Defines the curve boot strapping instrument type name. Details will be explained later.
Tenor - Tenor of each curve boot strapping instrument. Details will be explained later.
ScaleFactor - The scaling adjustment to the 'Value' field. This scale factor will be multiplied to the value in 'Value' field to represent the final quote value to use.
Value - The market quoted price/value.

Detailed Specification of 'InstrumentType' and 'Tenor':
Curve_USD_OIS support following boot strapping instrument types, and accordingly 'Tenor' format for each type of instrument.
(1) CashDeposit - If the instrument type is 'CashDeposit', the tenor should usually be 'ON', 'TN' or 'SN'. Also, it can use the typical tenor, e.g. '3M'.
(2) CompoundIndexSwap - If the instrument type is 'CompoundIndexSwap', the tenor should use the typical tenor, e.g. '10Y'.

Curve_USD_Libor3M Specification

Market Data Type: 
Curve_USD_Libor3M

Description: 
The LIBOR 3M curve of USD currency

Market Data required headers:
MarketDataType - The market data object to construct with the market data.
Source - The field to store the name of market data source.
Revision - The field to store which price type of "Bid", "Ask" or "Mid" the market data is.
Currency - The currency name.
InstrumentType - Defines the curve boot strapping instrument type name. Details will be explained later.
Tenor - Tenor of each curve boot strapping instrument. Details will be explained later.
ScaleFactor - The scaling adjustment to the 'Value' field. This scale factor will be multiplied to the value in 'Value' field to represent the final quote value to use.
Value - The market quoted price/value.

Detailed Specification of 'InstrumentType' and 'Tenor':
Curve_USD_Libor3M support following boot strapping instrument types, and accordingly 'Tenor' format for each type of instrument.
(1) CashDeposit - If the instrument type is 'CashDeposit', the tenor should use the typical tenor, e.g. '3M', '6M' etc. Also, it can use 'ON', 'TN' or 'SN'.
(2) FRA - If the instrument type is 'FRA', the tenor should follow one of the formats below:
	(a) Tenor = [Name]_[StartTenor]_[EndTenor], where [Name] is an arbitrary descriptive name, [StartTenor] and [EndTenor] are the start and end tenors of FRA contract, e.g. 'FRA_1M_7M' for the FRA contract with start tenor '1M' and end tenor '7M'.
	(b) Tenor = [Name]_[StartDate]_[EndDate], where [Name] is an arbitrary descriptive name, [StartDate] and [EndDate] are the start and end dates of FRA contract, e.g. 'FRA_18-Sep-2019_18-Dec-2019' for the FRA contract with start date '18-Sep-2019' and end date '18-Dec-2019'.
(3) STIRFuture - If the instrument type is 'STIRFuture', the tenor should use the future contract month code starting with month code followed by two-digit year code, e.g. 'U19' for September 2019.
(4) Swap - If the instrument type is 'Swap', the tenor should use the typical tenor, e.g. '10Y'.

FX Spot Matrix Specification

FX Spot Matrix so far only supports USD as the intermediate exchange rate quote currency:
(1) USD - FXSpot_USD
(Later more intermediate exchange rate quote currencies may be subject to add.)

Any FX spot matrix will be following the same market data specification. We take 'Curve_USD_Implied_USDCNY' as an example for details.

---------------------------------

FXSpot_USD Specification

Market Data Type: 
FXSpot_USD

Description: 
FX Spot matrix constructed by market data that all currencies exchange rates are quoted in USD.

Market Data required headers:
MarketDataType - The market data object to construct with the market data.
Source - The field to store the name of market data source.
Revision - The field to store which price type of "Bid", "Ask" or "Mid" the market data is.
Currency - The currency name of the exchange rate.
Inversed - Indicate whether the quoted exchange rate in the 'Value' field is a rate of '1-unit' base currency in currency ('FALSE') or a rate of '1-unit' currency in base currency('TRUE'). e.g. For 'FXSpot_USD' it indicate whether the quoted exchange rate is target currency to USD rate ('FALSE') or USD to target currency rate('TRUE').
ScaleFactor - The scaling adjustment to the 'Value' field. This scale factor will be multiplied to the value in 'Value' field to represent the final quote value to use.
Value - The market quoted price/value.

Curve_USD_Implied_USDCNY Specification

Market Data Type: 
Curve_USD_Implied_USDCNY

Description: 
The implied discounting curve of USD currency, constructed by a CNY discounting curve and a series of USDCNY forward exchange rates.

Market Data required headers:
MarketDataType - The market data object to construct with the market data.
Source - The field to store the name of market data source.
Revision - The field to store which price type of "Bid", "Ask" or "Mid" the market data is.
BaseCurrency - The base currency name.
Currency - The currency name.
InstrumentType - Defines the curve boot strapping instrument type name. Details will be explained later.
Tenor - Tenor of each curve boot strapping instrument. Details will be explained later.
Inversed - Indicate whether the quoted exchange rate in the 'Value' field is a rate of '1-unit' base currency in currency ('FALSE') or a rate of '1-unit' currency in base currency('TRUE').
ScaleFactor - The scaling adjustment to the 'Value' field. This scale factor will be multiplied to the value in 'Value' field to represent the final quote value to use.
Value - The market quoted price/value.

Detailed Specification of 'InstrumentType' and 'Tenor':
Curve_USD_Implied_USDCNY support following boot strapping instrument types, and accordingly 'Tenor' format for each type of instrument.
(1) FXSpot - If the instrument type is 'FXSpot' the tenor only accept 'SP'. Note: 'FXSpot' is a required quote for every FX forward implied discounting curve. It is required and only one is accepted.
(2) FXForwardPoint - If the instrument type is 'FXForwardPoint', the tenor should use 'ON', 'TN', 'SN' and typical tenors like '1W', '3M', '1Y' etc.
(3) FXForwardOutright - If the instrument type is 'FXForwardOutright', the tenor should use typical tenors like '1W', '3M', '1Y' etc. It does not support 'ON', 'TN', 'SN' tenors

Note: 'FXForwardOutright' does not support 'ON', 'TN', 'SN' tenors; if need to provide forward rates on 'ON', 'TN' and 'SN' tenors, convert market data and switch to use 'FXForwardPoint' instrument type.

---------------------------------

Vol_USDCNY_OptionVol Specification

Market Data Type: 
Vol_USDCNY_OptionVol

Description:
The Black-implied volatility surface of USDCNY exchange rate European vanilla options.

Market Data required headers:
MarketDataType - The market data object to construct with the market data.
Source - The field to store the name of market data source.
Revision - The field to store which price type of "Bid", "Ask" or "Mid" the market data is.
BaseCurrency - The base currency name.
Currency - The currency name.
InstrumentType - Always be 'EuropeanOption'. Defines the instrument type of the market volatility quote.
Tenor - Tenor of each market quote instrument (e.g. European option), typical tenors like '1W', '3M', '1Y' etc. Cannot be 'ON', 'TN' or 'SN'.
BFRR - The type name for strike level. It takes ATM or the representation of a delta name followed by BF/RR. For example, 'ATM', '25D BF', '25D RR', '10D BF', '10D RR' etc.
ScaleFactor - The scaling adjustment to the 'Value' field. This scale factor will be multiplied to the value in 'Value' field to represent the final quote value to use.
Value - The market quoted price/value.

Curve_WTI_CmdtyForward Specification

Market Data Type: 
Curve_WTI_CmdtyForward

Description:
The forward price curve constructed from prices of a series of Future contracts.

Market Data required headers:

Detailed Specification of 'InstrumentType' and 'Contract':
Curve_WTI_CmdtyForward support following commodity instrument types for forward curve construction, and accordingly 'Contract' naming rules for each type of instrument.
(1) Future - If the instrument type is 'Future', the naming rule of 'Contract' field should be 'Contract' = [ContractName]_[LastTradeDate], where [ContractName] is an arbitrary descriptive name, [LastTradeDate] is the last date when the future contract is available for trade, e.g. 'CLN9 Comdty_20-Jun-2019' for the CLN9 Comdty contract which is available to trade util the last trade date on '20-Jun-2019'. The accuracy of [LastTradeDate] field is critical in building an accurate commodity forward curve.
